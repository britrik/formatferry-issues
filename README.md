## About This Repo

- **Source code:** The Format Ferry codebase is private (this repo does **not** contain source code)
- **Purpose:** Public bug reports, feature requests, and community support
- **Response time:** We aim to respond within 48 hours
- **Status tracking:** All issues are tracked here publicly; internal fixes happen in the private repo

## Before You Report

1. **Check existing issues:** Search [open issues](../../issues) first
2. **Include details:** OS, browser, file format, error messages, screenshots
3. **Be specific:** Steps to reproduce, expected vs actual behaviour
4. **One issue per report:** Don't bundle multiple bugs in one issue

## Security

🔒 **Do not report security vulnerabilities publicly.**

Email security issues directly to: `security@vibingfun.com`

We respond to security reports within 24 hours and coordinate disclosure.

## FAQ

**Q: Why is the source code private?**
> Format Ferry's core is proprietary. We maintain a public issue tracker so users can report problems without needing repo access.

**Q: Can I contribute code?**
> Not directly — the repo is closed-source. But feature requests are welcome, and we credit community ideas in release notes.

**Q: How do I know if my issue was fixed?**
> Watch this repo for updates. We close issues when fixes deploy to [formatferry.vibingfun.com](https://formatferry.vibingfun.com).
